Layout Module

The layout module allows to easily create a reactive web-app layout working across desktop and mobile. It consists of <ea-layout> , <ea-toolbar>, <ea-main>, <ea-body> , <ea-drawer>, <ea-footer>, and the LayoutService.

Layout Module

<ea-toolbar>

<ea-toolbar> has a button that is dynamically generated based off whether or not there is currently in <ea-drawer> initialized in the application.

<ea-toolbar>

<ea-drawer>

The ea-drawer is designed to have a ,<header>, <section>, and <footer>. A <header> element will always sit at the top of the drawer. The <section> is stretched to be the space between the header and the footer aligned at the top. The <footer> is aligned at the bottom always. For simplicity sake we accomplish this we flexbox which makes it ideal to only have 3 elements, header, section, and footer to maintain this layout easily.

<ea-drawer>

The drawer uses different an event stream to inform the application what state it is in. The first two states we've introduced are maximized and closed. We plan to introduce new states to make it so that you can adjust the size of your drawer to different states we think have utility.
