Themeing

Styling and theming in eang is fully customizable and can be easily changed.

Unlike other component frameworks, the style is not set in stone and does not abstract or encapsulates away the power of CSS. Instead it embraces the new capabilities of CSS and puts design back into the hand of the designer.

New and upcoming features of CSS eliviate a lot of the maintainability problems of CSS. eang relies heavily on CSS Custom Properties and nesting and uses a PostCSS pipeline to support older browsers.

eang has multiple build-in themes. Each theme can be extensively customized with CSS custom properties. Each theme has global variables as well as component based CSS variables.

These properties can also just apply within a specific scope of your page.

If overriding variables and properties for some reason does not work in your case, there is nothing prevents you to create your own theme. You can integrate

If that is not enough, eang allows you to define the CSS of each component individually additional themes can be be added. This allows users of eang to fully customize the the look and feel of each component and makes it easy to comply with Coporate Design guidelines. eang embraces CSS and takes advatage of upcoming CSS features. PostCSS is used to support all older browsers.
